# 月次経理(個人事業主)

teachAI を個人事業として運営する前提の月次クローズ手順。

## 標準仕訳パターン

### A. SaaS 費用(国内・円建て)
```
通信費 or 支払手数料  / 普通預金
(摘要: Vercel Pro 2026/05)
```
- Vercel Pro / Supabase Pro / GitHub: 「通信費」or「支払手数料」で統一
- LLM API(月額固定なし、従量): 「通信費」or 専用に「研究開発費」

### B. SaaS 費用(海外・外貨)
```
通信費  XXX 円(月平均レートで換算)/ クレジットカード
(摘要: Vercel Pro $20 @ 156円 = 3,120円)
```
- 月平均 TTM レートで換算(国税庁公表値)
- 為替差損益は決算時にまとめて処理

### C. LLM API 費用(従量課金)
```
研究開発費 XXX / クレジットカード
(摘要: Anthropic API 2026/05)
```
- 量が大きくなったら別科目「外注費」or「業務委託費」検討
- 個人利用と業務利用が混在する場合は按分(例: 80% 業務)

### D. ドメイン・サーバー
```
通信費 XXX / 普通預金
(摘要: ドメイン更新 1年分 → 短期前払費用化を検討)
```

### E. 売上(個人プラン)
```
売掛金 or 普通預金  XXX / 売上高
(消費税課税事業者の場合は 仮受消費税 を分離)
```

### F. 売上(ライフタイム)
```
普通預金  XXX / 前受収益(長期)
毎月: 前受収益 / 売上高(償却期間で按分)
```
- 「ライフタイム」だが、税法上の償却期間を税理士と相談
- 6 ヶ月での L1 黒字化目標(¥2.5M lifetime + ¥10万 MRR)を意識

## 月次クローズ手順

1. クレジットカード明細を取り込み(CSV)
2. SaaS 費用を A-D に分類
3. Stripe / 売上明細から E-F を入力
4. 月平均 TTM レートで外貨換算
5. 仮払消費税 / 仮受消費税 のチェック
6. 試算表を出力
7. 異常値(前月比 +50% 以上)を確認

## 出力フォーマット

```
# 月次レポート YYYY/MM

## 売上
- 個人プラン MRR: XXX
- ライフタイム新規(月按分後): XXX
- 合計売上: XXX

## 費用
- 通信費 (SaaS): XXX
- 研究開発費 (LLM API): XXX
- その他: XXX
- 合計: XXX

## 営業損益
XXX

## 異常値
- ...

## 来月の留意点
- ...
```

## 注意

- 個人事業主 → 法人成りの判断は売上高・所得・社会保険の総合判断
- 消費税の課税事業者選択は慎重に(インボイス影響あり)
- 税理士に渡せる程度の粒度で月次クローズしておく

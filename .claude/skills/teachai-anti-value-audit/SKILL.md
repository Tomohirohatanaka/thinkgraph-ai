# teachAI アンチバリュー監査

teachAI の北極星「人間の思考力を取り戻す」を守るための、機械的な却下フィルタ。

## 9つのアンチバリュー(§3.4)

監査対象の提案 / コード / コピーが、以下のいずれかに該当するかをチェックする。

| # | アンチバリュー | 該当例 |
|---|---|---|
| 1 | 答えを直接出す機能 | 回答生成、要約代行、調べ物API、「正解はこれ」表示 |
| 2 | 学習効率の最大化UX | 「最短で覚える」「効率的に習得」訴求、スキマ時間特化UI |
| 3 | スコア順ランキング・煽り通知 | リーダーボード、「あなたは下位X%です」、緊急通知 |
| 4 | 中毒性UX | 無限スクロール、デイリーストリーク強要、コンボボーナス |
| 5 | 注意経済の搾取 | 広告掲載、行動追跡型データ販売、第三者トラッカー |
| 6 | ユーザーの代わりに考える | 意思決定の代行、自動最適化、おすすめのコース提案 |
| 7 | 法人向けの強制利用販売 | 企業研修義務化、「全社員が使う」前提のエンプラ販売 |
| 8 | 子供向け展開 | 18歳未満ターゲット、学校現場の児童利用 |
| 9 | ダークパターン | 解約困難化、誤解を招く価格表示、自動更新の隠蔽 |

## 監査プロセス

1. **入力を受け取る** — 提案文 / コード / コピー / 機能仕様 / 価格表
2. **9項目を順番にチェック** — 各項目について「該当 / 該当しない / 判断保留」で答える
3. **該当または保留がある場合** — 却下し、該当する番号と理由を出力
4. **すべて該当しない場合** — 通過と判定し、次の判断ステップ(`polaris-5q`)に進めるべきと提案

## 出力フォーマット

```
# アンチバリュー監査結果

提案: <提案の要約 1 行>

| # | 項目 | 判定 | 根拠 |
|---|---|---|---|
| 1 | 答えを直接出す | 通過 / 該当 / 保留 | ... |
| ... | ... | ... | ... |

結論: 通過 / 却下 / 要改修
推奨アクション: <次に何をすべきか>
```

## 判定の厳しさ

- 「ちょっとだけ」「条件付き」「オプションで」は **該当扱い** とする。
- 「他社もやっている」「ユーザーが要望している」「短期的に売上が立つ」は **判定を覆さない**。
- 該当が 1 つでもあれば却下。半端な妥協は北極星を毀損する。

## 通過例

- 「ユーザー自身が書いた説明を再読する促し通知」 → 通過(中毒性ではなく主体的な検算促進)
- 「教材原文の引用を伴うスコア説明」 → 通過(説明可能性を高める)

## 却下例

- 「3日連続使用バッジ」 → 該当 #4(中毒性UX)
- 「他のユーザーが学んでいる人気トピック」 → 該当 #3(ランキング誘導)
- 「教える相手を AI が自動選定」 → 該当 #6(ユーザーの代わりに考える)
- 「企業の全社員向け一括契約プラン」 → 該当 #7(強制利用販売)
- 「キャンセル時に小さな文字で引き止め質問を 3 画面挟む」 → 該当 #9(ダークパターン)

## このスキルを呼ぶタイミング

- 新機能を実装するか議論するとき
- 価格表やプラン構成を変更するとき
- マーケコピーや LP のドラフトができたとき
- BD/法人案件の提案を受けたとき
- いかなる「ユーザーの行動を変えるUX」を入れるとき

このスキルは **却下を恐れない**。北極星を守るのが役目で、機能を実装するのは別の仕事。

# X 投稿草案

teachAI のブランド戦略(§16)に沿った投稿草案。

## ブランドトーン

- **思想を語る:** 機能ではなく「なぜ作るか」を中心に
- **謙虚な確信:** 「これが最強」ではなく「こう考えています」
- **数字より気づき:** 「MAU 1000 達成!」より「今週ユーザーから "AI に頼りすぎていた" と言われた」
- **検算文化:** 「学ぶ」ではなく「検算する」「思考力を取り戻す」

## NG 表現

- 「最強の学習ツール」「効率 X 倍」(§3.4#2 違反)
- 「3 日で覚える」「最短ルート」(同上)
- 「ライバルに差をつける」(知性の競争化、§3.4#3 違反)
- 「みんな使ってる」(同調圧力)
- 「今だけ X% OFF」(緊急性煽り、§3.4#9 抵触)

## OK 表現

- 「自分の理解の浅さに気づく時間を、週に何分か取り戻す」
- 「ChatGPT があれば答えはすぐ手に入る。でも 'なぜ知っているか' は手に入らない」
- 「ユーザーが教える側に回ると、知識の質感が変わる」
- 「今日、評価エンジンの説明可能性を一段強化しました。スコアの内訳が見えるようになりました」

## テンプレート

### A. 開発日記(日次)
```
今日やったこと:
- <技術的変更>
- <UX 改善>

得られた気づき:
<1-2行>

次にやること:
<1行>

#BuildInPublic #teachAI
```

### B. 機能告知
```
<機能名> をリリースしました。

これは <ユーザーが体験する変化> のためのもので、
<なぜ作ったか> です。

§3.4 アンチバリューチェック済み:
- 答えを出さない
- ランキング化しない
- 中毒性なし

詳しくは → <note URL>
```

### C. ユーザーの声(本人許諾済み)
```
ユーザーの方から
「<本人の言葉そのまま>」
というフィードバックをいただきました。

私たちが届けたいのはまさにこれです。
<簡潔な解説 1-2行>
```

### D. 価値発信
```
<問題提起 1行>

<解像度を上げる視点 2-3行>

<teachAI との接続 1-2行、過度に押し売らない>
```

## 作成プロセス

1. 投稿の目的を明確化(告知 / 日記 / 価値発信 / フィードバック紹介)
2. 適切なテンプレ選択
3. ドラフト作成
4. `teachai-anti-value-audit` を内部で実行(セルフチェック)
5. NG 表現リストと照合
6. 文字数調整(140-280 文字目安)

## 出力フォーマット

```
# X 投稿草案

種別: <A/B/C/D>
目的: ...

## 草案(候補1)
<本文>
文字数: X

## 草案(候補2)
<本文>
文字数: X

## セルフチェック
- アンチバリュー: 通過
- NG 表現: なし
- ブランドトーン整合: ...

推奨投稿時刻: <平日朝 / 夜 / 任意>
```

## 重要

X はバズらせるための場ではなく、「思想を共有する場」と位置づける。フォロワー数より、共鳴の質を優先する。

# 特許請求項ドラフト

teachAI のコア技術を仮出願に向けて文章化するスキル。

## 出願候補(優先順)

| 優先 | テーマ | 主な発明特徴 |
|---|---|---|
| 1 | 検算指向の評価エンジン | SOLO + RQS + 知識構築 を統合したスコア計算 |
| 2 | 知識グラフを介した忘却曲線 | グラフのエッジ強度と SRS スケジューラの連動 |
| 3 | キャラクター個性付き対話 UX | 同一スコア結果を 5 種類のキャラクター口調で返す表現多様性 |
| 4 | マルチ LLM Adapter | API キー形式自動判定・ルーティング(優先度低) |

## 請求項テンプレ

### 独立クレーム(請求項1)
```
【請求項1】
ユーザーが教材に基づいて他者(またはAIキャラクター)に説明を行う対話において、
当該説明を <要素A>、<要素B>、<要素C>の少なくとも3つの観点から評価するシステムであって、

<要素A>: 構造の深さを示す SOLO レベル(1-5)
<要素B>: 質問の連鎖を示す RQS スコア
<要素C>: 知識グラフ上の新規ノード/エッジ生成数

を統合した複合スコアを生成し、
当該スコアに対応する教材引用箇所をユーザーに提示する、
教育支援システム。
```

### 従属クレーム(2 以降)
```
【請求項2】
請求項1において、複合スコアの算出にあたり、
<要素A>、<要素B>、<要素C>の重みを Elo レーティングに基づいて動的に変化させる、教育支援システム。

【請求項3】
請求項1において、生成された複合スコアと教材引用箇所を、
複数の事前定義された性格を持つキャラクターの口調で表現する出力モジュールを備える、教育支援システム。

【請求項4】
請求項1において、知識グラフのエッジ強度に基づき、
ユーザーごとに次回検算の推奨時期を算出する忘却曲線スケジューラを備える、教育支援システム。

【請求項5】
請求項1〜4のいずれかに記載のシステムを実現するための、コンピュータプログラム。
```

## 明細書アウトライン

```
1. 技術分野
   - 教育技術、対話 AI、評価アルゴリズム
2. 背景技術
   - 既存の AI チューター(Khanmigo 等)
   - SOLO Taxonomy(Biggs & Collis 1982)
   - SRS(Anki, SuperMemo)
3. 発明が解決しようとする課題
   - 「ユーザーが教える側」の評価メトリクスが既存に存在しない
   - 説明可能性の欠如
4. 課題を解決するための手段
   - 上記独立クレームの構成
5. 発明の効果
   - 検算文化の支援
   - 説明可能性の担保
6. 図面の簡単な説明
7. 発明を実施するための形態
   - コードレベルの実装例
   - 数式・閾値・パラメータ範囲
8. 産業上の利用可能性
9. 符号の説明
```

## 先行技術調査チェックリスト

- J-PlatPat(国内特許検索)で キーワード "教育" + "評価" + "対話" を検索
- Google Patents で同等キーワード(英語)
- 学術論文(Google Scholar)で SOLO + automatic + scoring
- 既存サービス(Khanmigo、Duolingo Max、Synthesis 等)の特許出願状況

# note 記事草案

teachAI の思想を伝える長文コンテンツ。

## 記事タイプ

| 種別 | 主旨 | 推奨文字数 |
|---|---|---|
| 思想エッセイ | 検算文化、思考主権など概念解説 | 2,000-4,000 |
| 開発裏話 | 機能の背景、設計判断の理由 | 1,500-3,000 |
| 月次レポート | KPI ではなく "学び" を中心に | 2,000-3,500 |
| ユーザーケース | 本人許諾済みの体験談+考察 | 2,000-4,000 |
| 競合との位置取り | ChatGPT 等との違い・補完関係 | 3,000-5,000 |

## 構成テンプレ

```
# <タイトル>

## はじめに(問題提起、200-400字)
読者が「自分のこと」と思える違和感の言語化。

## 本論1(現状の解像度を上げる、500-1000字)
「なぜこれが問題か」を 1 段深く。
データやエピソードを 1 つ。

## 本論2(別の角度、500-1000字)
逆説、対比、歴史的視点など。
「答えはまだ示さない」。

## 答えの提示(500-1500字)
ここで teachAI の思想を出す(機能ではなく考え方)。
3 つのポイントなど構造化。

## 限界の明示(200-500字)
「これはこういう時には向かない」を率直に書く。
信頼を担保する。

## 結び・行動への誘い(200-400字)
押し売りなし。読者が次にやれる小さい行動 1 つ。

---
P.S. teachAI を試してみたい方は <URL>(過剰に煽らない)
```

## 必須要素

- **限界の明示** — 完璧な道具ではないことを書く。誠実さの担保
- **数字の禁欲** — 「劇的に伸びた」より「○月○日にユーザー△人が"気づいた"と言ってくれた」
- **引用の作法** — 学術的引用は元論文へ。ニュースは一次情報へ
- **法令配慮** — 医療・教育・労働関連の表現は §20 リーガル戦略を確認

## NG

- 「AI 学習革命」「最強」「これだけで」(§3.4#2 違反)
- 競合の悪口
- ユーザーの個人体験を本人未確認で書く
- アフィリエイト混入

## 作成プロセス

1. テーマ確定
2. 構成テンプレからアウトライン
3. 各セクションの draft
4. `anti-value-guardian` で全文セルフチェック
5. 「10 年後にも価値が残るか」セルフレビュー
6. SEO 観点(タイトル・見出し)を最後に調整

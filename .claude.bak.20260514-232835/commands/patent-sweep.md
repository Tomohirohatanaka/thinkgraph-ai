# /patent-sweep

知財観点での棚卸しを実行します。

## 実行内容

1. **patent-strategist サブエージェントを呼ぶ**
2. 直近 1-3 ヶ月のコード変更・新機能を確認
3. 出願候補があれば `patent-claim-draft` スキルで請求項案
4. 先行技術調査計画
5. 公開前リスクチェック(発信予定の X 投稿・note 記事に発明開示が含まれていないか)

## 使い方

```
/patent-sweep [期間: 例 "2026Q1" / "last-90-days"]
```

## 出力

patent-strategist のフォーマットを参照。

## 重要

仮出願は権利化のためではなく「優先日確保」のため。
出願前に公の場で発明開示すると新規性を失う可能性があるので、四半期に 1 回は必ず実施。

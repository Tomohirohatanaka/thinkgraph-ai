# /audit-feature

新機能の採否を多段審査で決めます。

## 実行内容

1. **`teachai-feature-spec` スキル** — テンプレに沿った仕様書ドラフト生成
2. **`anti-value-guardian` サブエージェント** — §3.4 と判断5問で監査
3. **(通過時のみ)`cold-consultant` サブエージェント** — ユニットエコノミクス・実行性を含む経営判断
4. 結論(採用 / 改修要 / 却下)

## 使い方

```
/audit-feature <機能名> [機能の概要]
```

例:
```
/audit-feature 学習履歴のグラフ可視化機能
```

## 出力

```
# Feature Audit Result

機能名: ...
仕様書: <ファイルパス>

## アンチバリュー監査
- 結論: 通過 / 却下
- 詳細: ...

## Polaris 5Q
- Yes 数: X/5
- 結論: ...

## Cold Consultant
- 結論: ...
- 代替案: ...

## 最終判定
採用 / 改修要 / 却下
次のアクション: ...
```

## 重要

このコマンドは「やる前に止める」ためのもの。
通過後すぐ実装するのではなく、仕様書を pinned issue として 1-2 日寝かせるとさらに良い判断ができる。
